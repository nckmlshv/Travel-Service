# POST /trips/{tripId}/conflicts/{conflictId} → Сервис «Планировщик»
Спецификация обработки запроса POST /trips/{tripId}/conflicts/{conflictId} в сервисе «Планировщик».<br>

## Назначение
Разрешение конфликта синхронизации по поездке с применением выбранной стратегии и обновлением основной версии плана поездки.

## Входные данные

| Поле          | Тип данных | Обязательное | Ограничения / Описание |
|---------------|------------|--------------|------------------------|
| tripId        | string     | Да           | Path-параметр, UUID, идентификатор поездки |
| conflictId    | string     | Да           | Path-параметр, UUID, идентификатор конфликта |
| strategy      | string     | Да           | keep_server / keep_client / keep_both / manual_merge |
| resolvedBy    | string     | Да           | UUID, идентификатор пользователя, разрешившего конфликт |
| mergedPayload | object     | Нет          | Обязателен при strategy = manual_merge. Содержит итоговую версию данных |

### Пример запроса

    {
      "strategy": "manual_merge",
      "resolvedBy": "6f1c8b4d-3b6d-4c2b-a0bb-9d8c4e9f1234",
      "mergedPayload": {
        "title": "Поездка в Стамбул",
        "items": [
          {
            "itemId": "1db8c0ae-6f0e-4a98-bf86-0f53b2d31f91",
            "type": "flight",
            "status": "selected"
          }
        ]
      }
    }

## Выходные данные

| Поле        | Тип данных | Обязательное | Описание |
|-------------|------------|--------------|----------|
| conflictId  | string     | Да           | UUID конфликта |
| status      | string     | Да           | Статус результата обработки, resolved |
| resolution  | string     | Да           | Примененная стратегия разрешения конфликта |
| tripVersion | integer    | Нет          | Новая версия поездки после разрешения конфликта |
| message     | string     | Нет          | Сообщение о результате выполнения |

### Пример ответа

    {
      "conflictId": "8a6d2d30-8d80-4f4f-a2d4-7fcb6b4fd001",
      "status": "resolved",
      "resolution": "manual_merge",
      "tripVersion": 12,
      "message": "Конфликт успешно разрешен"
    }

## Алгоритм работы сервиса «Планировщик»

1. Проверка доступа:
    1. Проверить авторизацию пользователя и права на изменение поездки<br>
       - Пользователь не авторизован → Прекратить выполнение
       - У пользователя нет прав на разрешение конфликта → Прекратить выполнение
       - Проверка пройдена → Продолжить выполнение
2. Проверка существования сущностей:
    1. Найти поездку по tripId<br>
       - Поездка не найдена → Прекратить выполнение
       - Поездка найдена → Продолжить выполнение
    2. Найти конфликт по conflictId в рамках указанной поездки<br>
       - Конфликт не найден → Прекратить выполнение
       - Конфликт найден → Продолжить выполнение
    3. Проверить статус конфликта<br>
       - Конфликт уже разрешен другим участником → Прекратить выполнение
       - Конфликт открыт → Продолжить выполнение
3. Валидация данных:
    1. Проверить входные данные на соответствие типам данных и ограничениям
       - Хотя бы одно из передаваемых полей не валидно → Прекратить выполнение
       - Передаваемые данные валидны → Продолжить выполнение
    2. Проверить наличие mergedPayload для стратегии manual_merge
       - strategy = manual_merge и mergedPayload отсутствует → Прекратить выполнение
       - Условие выполнено → Продолжить выполнение
4. Применение стратегии разрешения:
    1. В зависимости от значения strategy выполнить одно из действий<br>
       - keep_server → Сохранить серверную версию данных как итоговую
       - keep_client → Применить клиентскую версию данных как итоговую
       - keep_both → Сохранить обе версии в итоговом плане поездки
       - manual_merge → Применить mergedPayload как итоговую версию данных
5. Обновление состояния:
    1. Обновить основную версию плана поездки
    2. Установить для конфликта статус resolved
    3. Сохранить информацию о пользователе, выполнившем разрешение конфликта
6. Вернуть ответ 200 OK

## Ошибки

### Формат ошибки

| Поле    | Тип данных | Описание |
|---------|------------|----------|
| code    | string     | Статус-код ошибки |
| message | string     | Описание ошибки |
| details | object     | Дополнительные данные об ошибке |

### Пример ошибки

    {
      "code": "CONFLICT_ALREADY_RESOLVED",
      "message": "Конфликт уже разрешен другим участником",
      "details": {
        "conflictId": "8a6d2d30-8d80-4f4f-a2d4-7fcb6b4fd001"
      }
    }

## Возможные ошибки

| Код | Описание |
|-----|----------|
401 | Пользователь не авторизован  
403 | Недостаточно прав для выполнения операции  
404 | Поездка или конфликт не найдены  
409 | Конфликт уже разрешен другим участником  
422 | Ошибка валидации входных данных
