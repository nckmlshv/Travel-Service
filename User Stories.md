# User Stories

### US-01. Как участник группы я хочу видеть общий актуальный план поездки, чтобы понимать текущее согласованное состояние маршрута  
  
Критерии приемки:  
- Отображаются все выбранные элементы поездки  
- Отображаются предложенные альтернативы по элементам  
- Видно, какой вариант сейчас является основным  

### US-02. Как участник группы я хочу добавлять рейс, отель, трансфер или активность, чтобы формировать общий маршрут поездки  
  
Критерии приемки:  
- Можно добавить элемент одного из поддерживаемых типов  
- Новый элемент появляется в общем плане поездки  
- Остальные участники видят добавленный элемент после синхронизации  

### US-03. Как участник группы я хочу предлагать альтернативный вариант отеля, рейса или активности, чтобы группа могла выбрать лучший вариант без потери текущего решения  
  
Критерии приемки:  
- Альтернативный вариант добавляется отдельно от основного  
- Текущий выбранный вариант не перезаписывается  
- Все участники видят список альтернатив  

### US-04. Как организатор поездки я хочу утверждать один из предложенных вариантов, чтобы фиксировать итоговое решение для всей группы  
  
Критерии приемки:  
- Организатор может выбрать один вариант основным  
- Предыдущий основной вариант переводится в альтернативные или снимается с выбора  
- Всем участникам отображается обновлённый основной вариант  

### US-05. Как участник группы я хочу редактировать существующий элемент поездки, чтобы актуализировать маршрут при изменении условий поездки  
  
Критерии приемки:  
- Можно изменить параметры элемента  
- Изменения сохраняются в системе  
- Другие участники видят обновлённые данные после синхронизации  

### US-06. Как участник группы я хочу работать с планом поездки оффлайн, чтобы не терять возможность вносить изменения без интернета  
  
Критерии приемки:  
- Пользователь может просматривать ранее загруженный план без сети  
- Пользователь может вносить изменения оффлайн  
- После восстановления связи изменения отправляются на синхронизацию  

### US-07. Как участник группы я хочу видеть, что мои изменения ещё не синхронизированы, чтобы понимать, что другие участники пока не видят их  
  
Критерии приемки:  
- Для локальных изменений отображается статус несинхронизированности  
- После успешной отправки статус исчезает или меняется на подтверждённый  
- Пользователь может отличить локальные изменения от общих  

### US-08. Как участник группы я хочу получать уведомление о конфликте, если другой пользователь одновременно изменил тот же элемент, чтобы выбрать корректную версию  
  
Критерии приемки:  
- Система выявляет конфликт конкурентных изменений  
- Пользователю показываются конфликтующие версии  
- Пользователю доступен выбор варианта разрешения конфликта  

### US-09. Как участник группы я хочу видеть историю изменений элемента поездки, чтобы понимать, кто и когда изменял маршрут  
  
Критерии приемки:  
- Для элемента доступен список изменений  
- Для каждого изменения отображаются автор и время  
- Видно, какие данные были изменены  

### US-10. Как участник группы я хочу получать уведомления об изменениях маршрута и внешних событиях, чтобы своевременно реагировать на проблемы в поездке  
  
Критерии приемки:  
- Уведомления приходят при изменениях плана другими участниками  
- Уведомления приходят при конфликтах, задержках и изменениях бронирований  
- Уведомление содержит краткое описание события
