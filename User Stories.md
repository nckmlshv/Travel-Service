# User Stories

**1. Как пользователь группы я хочу видеть общий план поездки, чтобы понимать актуальное состояние**    
  
Критерии приемки:  
- Отображаются все элементы поездки (рейсы, отели, активности)  
- Отображается текущий выбранный вариант по каждому элементу  
- Данные синхронизированы с сервером  

**2. Как пользователь я хочу добавлять элементы поездки, чтобы формировать маршрут**  
  
Критерии приемки:  
- Можно добавить рейс/отель/активность/трансфер  
- Элемент появляется в общем плане  
- Все участники видят добавленный элемент  

**3. Как пользователь я хочу редактировать элементы поездки, чтобы обновлять план**  
  
Критерии приемки:  
- Можно изменить параметры элемента  
- Изменения сохраняются  
- Изменения отображаются у всех участников  

**4. Как пользователь я хочу удалять элементы поездки, чтобы корректировать маршрут**  
  
Критерии приемки:  
- Элемент можно удалить  
- Удаление отражается у всех участников  
- Удаление не приводит к потере истории  

**5. Как пользователь я хочу предлагать альтернативные варианты, чтобы группа могла выбирать**

Критерии приемки:  
- Можно добавить альтернативный вариант к элементу  
- Текущий вариант не перезаписывается  
- Все участники видят альтернативы  

**6. Как пользователь я хочу выбирать основной вариант из предложенных, чтобы зафиксировать решение**  
  
Критерии приемки:  
- Можно выбрать один вариант как основной  
- Остальные остаются как альтернативы  
- Выбор виден всем участникам  

**7. Как пользователь я хочу получать изменения от других участников, чтобы быть в курсе**
  
Критерии приемки:  
- Изменения синхронизируются автоматически  
- Пользователь видит обновления без перезагрузки  
- Отображается, кто внес изменение  

**8. Как пользователь я хочу работать оффлайн, чтобы не терять возможность редактирования**
  
Критерии приемки:  
- Можно просматривать сохранённый план без сети  
- Можно вносить изменения оффлайн  
- Изменения синхронизируются при восстановлении соединения  

**9. Как пользователь я хочу, чтобы система обрабатывала конфликты изменений, чтобы не терять данные**  
  
Критерии приемки:  
- Система выявляет конфликт изменений  
- Пользователю предлагаются варианты решения (merge)  
- Ни одно изменение не теряется  

**10. Как пользователь я хочу получать уведомления об изменениях и проблемах, чтобы оперативно реагировать**  
  
Критерии приемки:  
- Уведомления приходят при изменениях плана  
- Уведомления приходят при конфликтах и изменениях внешних данных  
- Уведомления содержат краткое описание события
